# 使用说明

## 启动方式

### 方式一：一键启动（推荐）

**Windows 用户：**
1. 双击 `运行.bat` 文件
2. 脚本会自动完成以下操作：
   - 检测 Python 环境
   - 创建虚拟环境（首次运行）
   - 安装依赖包
   - 启动图形界面

**Linux/macOS 用户：**
1. 打开终端，进入项目目录
2. 运行命令：
   ```bash
   chmod +x 运行.sh
   ./运行.sh
   ```

### 方式二：手动启动

**使用虚拟环境（推荐）：**
```bash
# 创建虚拟环境（首次）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 启动应用
python app.py
```

**直接启动：**
```bash
# 安装依赖
pip install -r requirements.txt

# 启动图形界面
python app.py

# 或启动命令行版本
python cli.py --help
```

## 虚拟环境说明

### 什么是虚拟环境？

虚拟环境是 Python 的独立运行环境，可以：
- 隔离不同项目的依赖
- 避免版本冲突
- 保持系统 Python 环境干净

### 为什么使用虚拟环境？

**优点：**
- ✅ 依赖隔离：不影响系统 Python
- ✅ 版本管理：每个项目独立的依赖版本
- ✅ 易于清理：删除 venv 文件夹即可
- ✅ 便于部署：requirements.txt 记录所有依赖

**缺点：**
- ❌ 占用额外空间（约 20-50MB）
- ❌ 需要激活环境

### 虚拟环境管理

**创建虚拟环境：**
```bash
python -m venv venv
```

**激活虚拟环境：**
```bash
# Windows
venv\Scripts\activate

# Linux/macOS
source venv/bin/activate
```

**退出虚拟环境：**
```bash
deactivate
```

**删除虚拟环境：**
```bash
# 直接删除 venv 文件夹
# Windows
rmdir /s venv

# Linux/macOS
rm -rf venv
```

## 常见问题

### Q: 运行脚本提示"未检测到 Python"？

**原因：**
- Python 未安装
- Python 未添加到 PATH 环境变量

**解决方案：**
1. 下载并安装 Python 3.7+：https://www.python.org/downloads/
2. 安装时勾选 "Add Python to PATH"
3. 重启命令行窗口

### Q: 虚拟环境创建失败？

**Windows：**
```bash
# 确保安装了完整的 Python
python -m pip install --upgrade pip
python -m venv venv
```

**Linux/macOS：**
```bash
# 安装 venv 模块
sudo apt install python3-venv  # Ubuntu/Debian
sudo yum install python3-venv  # CentOS/RHEL

# 创建虚拟环境
python3 -m venv venv
```

### Q: 依赖安装失败？

**可能原因：**
1. 网络连接问题
2. pip 版本过旧
3. 权限不足

**解决方案：**
```bash
# 升级 pip
python -m pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# Windows 以管理员身份运行
# Linux/macOS 使用 sudo（不推荐，建议使用虚拟环境）
```

### Q: 图形界面无法启动？

**可能原因：**
1. 缺少图形界面环境
2. tkinter 模块未安装
3. 远程 SSH 连接

**解决方案：**

**Windows：**
- tkinter 通常随 Python 一起安装
- 重新安装 Python，确保勾选 "tcl/tk and IDLE"

**Linux：**
```bash
# Ubuntu/Debian
sudo apt install python3-tk

# CentOS/RHEL
sudo yum install python3-tkinter

# Arch Linux
sudo pacman -S tk
```

**macOS：**
```bash
# 使用 Homebrew 安装的 Python 通常包含 tkinter
brew install python-tk
```

**SSH 远程连接：**
```bash
# 启用 X11 转发
ssh -X user@host

# 或使用命令行版本
python cli.py --auto --common
```

### Q: 如何卸载？

**完全卸载：**
```bash
# 1. 删除虚拟环境
rm -rf venv  # Linux/macOS
rmdir /s venv  # Windows

# 2. 删除项目文件夹
cd ..
rm -rf lan-port-scanner  # Linux/macOS
rmdir /s lan-port-scanner  # Windows
```

**保留配置卸载：**
```bash
# 只删除虚拟环境和缓存
rm -rf venv __pycache__ *.pyc
```

## 性能优化

### 提高扫描速度

1. **增加并发数**：100-200（默认 100）
2. **减少超时时间**：0.5-1.0 秒（默认 1.0）
3. **禁用隐蔽模式**：取消勾选隐蔽模式
4. **减少扫描范围**：只扫描必要的主机和端口

### 降低资源占用

1. **减少并发数**：20-50
2. **增加超时时间**：2.0-5.0 秒
3. **启用隐蔽模式**：添加随机延迟
4. **分批扫描**：将大范围扫描分成多次小范围扫描
